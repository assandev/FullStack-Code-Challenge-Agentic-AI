# Technical Support Guidelines

This document contains troubleshooting guidance for technical issues related to the banking platform, including login, app usage, and system errors.

---

## 1. Login Issues

Description:
Users may experience difficulty accessing their account.

### Common Causes
- Incorrect credentials
- Expired password
- OTP issues
- App not updated

### Support Guidance
- Ask the user to verify credentials
- Suggest password reset via "Forgot Password"
- Confirm OTP is entered correctly
- Ensure app is updated to latest version

### Escalation Signals
- User cannot recover access after multiple attempts
- Account appears locked
- OTP not being received persistently

---

## 2. Account Locked

Description:
Accounts may be locked after multiple failed login attempts.

### Support Guidance
- Inform the user that the lock is temporary (if applicable)
- Guide them through account recovery steps
- Suggest waiting period if lock is time-based

### Escalation Signals
- User cannot unlock account via recovery flow
- Identity verification fails
- User expresses urgency or frustration

---

## 3. Mobile App Errors

Description:
Users may encounter crashes, bugs, or unexpected behavior.

### Common Causes
- Outdated app version
- Device compatibility issues
- Network problems

### Support Guidance
- Ask user to update the app
- Suggest reinstalling the app
- Check internet connection
- Restart device

### Escalation Signals
- Persistent errors after troubleshooting
- App crashes blocking critical actions
- Multiple users reporting similar issue (system-wide problem)

---

## 4. OTP / Verification Issues

Description:
Users may not receive or be able to use verification codes.

### Support Guidance
- Confirm phone/email is correct
- Ask user to wait and retry
- Suggest checking spam or blocked messages

### Escalation Signals
- OTP not received after multiple attempts
- User cannot complete verification process
- Possible account access block

---

## 5. Slow or Failed Transactions (Technical)

Description:
User reports issues executing actions in the app.

### Support Guidance
- Confirm network connectivity
- Ask user to retry action
- Check if app is updated

### Escalation Signals
- Repeated failures
- System error messages
- Transactions stuck in pending without update

---

## General Rules for Support Agent

- Always provide clear step-by-step troubleshooting
- Start with simplest solution first
- Avoid technical jargon when possible
- If issue persists after 1–2 steps → consider escalation

---

## Tone Guidelines

- Be calm and helpful
- Guide the user step-by-step
- Acknowledge frustration when relevant
